💬 使い方

自然言語で指示するだけ。AIがトレイ確認→制約検証→変換→印刷→完了確認まで一貫実行。

ユーザー: この報告書をA4両面で左上ステープル、2穴パンチで10部印刷して AI: get_printer_status → idle, カセット1: A4普通紙 ✓ validate_print_options({Stpl:Front, Scnt:All, Pnch:2Hole, Duplex:DuplexNoTumble, PageSize:A4}) → ✅ 2,134ルール検査済み、違反0 report.docx → Graph API → PDF (245KB) print_document → Job #42 送信: 10部, A4両面, 左上ステープル, 2穴パンチ get_job_status(42) → ✅ completed
ユーザー: この企画書を中綴じ製本でA4冊子にして AI: get_printer_status → カセット3: A3普通紙 ✓ (中綴じA4はA3紙使用) validate_print_options({KCBooklet:Left, Fold:True, PageSize:A4}) → ✅ 有効 (20枚以内: 14枚=56p ✓) proposal.pptx → Graph API → PDF (1.2MB) print_document → Job #43 A4左綴じ中綴じ製本(A3紙使用→折り→A4冊子) → ✅ completed

🛡️ PPD制約バリデーション

Kyocera PPDファイルから抽出した3,192件のUIConstraints + cupsUIConstraintsルールで、印刷前にオプション互換性を自動検証。日本語エラーメッセージ付き。

❌ 中綴じ + コーナーステープル
ステープル(Front) と 中綴じ製本(Left) は同時に指定できません
❌ A5 + 二つ折り
折りモード(Bifold) と 用紙サイズ(A5) は同時に指定できません
❌ 厚紙 + パンチ
用紙種類(Cardstock) と パンチ(2Hole) は同時に指定できません
✅ A4両面 + ステープル + パンチ
全て有効。印刷できます。（2,134ルール検査済み、違反0）

10の重要ルール

1. 中綴じとステープルは同時不可（自動ステープル）
2. 折り使用時は OutputBin=FLDTRAY 必須
3. A5以下はステープル/折り不可
4. 封筒/ラベル/OHPは後処理全不可
5. 厚紙はステープル/パンチ/折り全不可
6. 中綴じPageSizeは仕上がりサイズ（A4→A3紙自動）
7. 三つ折りはA4/Letterのみ
8. 中綴じ最大20枚(80p)、折り最大3枚、普通紙のみ
9. 印刷前に validate_print_options で必ず検証
10. get_printer_status でトレイ紙設定を確認

🚀 セットアップ（5ステップ）

Vercelワンクリックではありません。Proxmox LXC + CUPS + Kyoceraドライバ + Cloudflare Tunnel の構成です。

Proxmox LXCコンテナ作成
Ubuntu 24.04テンプレートからクローン。2コア/4GB/20GB推奨。onboot=1で自動起動。
CUPS + Kyoceraドライバ + プリンタ登録
cups, cups-client, cups-filters, ghostscriptをインストール。Kyocera UPD v10.0のdebパッケージを導入。lpadminでsocket://プリンタIP:9100を登録。フィニッシャー設定（DF-7150, Punch, Fold, Inserter）。
MCPサーバーデプロイ
Node.js 22をインストール。リポジトリをgit clone → npm install → npm run build。systemdサービスを作成して環境変数（MCP_API_KEY, GRAPH_*）を設定。
Cloudflare Tunnel
cloudflaredをLXC内にインストール。Cloudflareダッシュボードでトンネル作成→トークン取得→cloudflared service install。hostname → http://localhost:3000 のingressルールを設定。
クライアント接続
Claude.ai → Settings → MCP → Add で URL: https://printer-mcp.your-domain/mcp?key=YOUR_KEY を設定。Claude Desktop/Cursor/Claude Codeも対応。
